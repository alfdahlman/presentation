## HTML

HTML, _Hyper Text Markup Language_, is a markup language used to structure content on the web. It defines elements using tags, like `<div>` or `<paragraph>`.
A HTML document consist of two main parts:

- `<head>`: Contains metadata, links to scripts and styles, and page info for the browser.
- `<body>`: Contains the actual content visible to the user.

A HTML document could be as simple as:

```
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>My first website</title>
  </head>
  <body>
    <h1>A heading...</h1>
    <p>...and a paragraph</p>
  </body>
</html>
```

Opening this file in a browser yields your very first webpage. :tada:

### DOM

When a browser loads an HTML document, it builds a tree-like structure in memory, called the **DOM** (Document Object Model).

The DOM is a live, dynamic representation of the HTML, allowing scripts to read and manipulate the page after its loaded.

> To inspect the DOM of a webpage, open the `Developer tools` in your browser, and select the `Elements` tab.

## CSS

**CSS** (Cascading Style Sheets), is used to style the HTML content. Modern browsers (mostly) interpret CSS the same way, allowing us to write styles that looks the same across different browsers.

To apply styles, you select elements using a _selector_, and define the _style_ to apply.

```
h1 {
    color: red;
}
```

In this example, all `<h1>` elements will have the color red.

If you need more control, you can use **classes**.

```
.red {
    color: red;
}
```

CSS reads from top to bottom, with the later rule winning. But it also uses a **specificity scoring system**. For example, selecting a `.class` is more specific than selecting a `tag`.

There are two common ways to include css:

## JavaScript

_JavaScript_ is a programming language which can run in the browser and lets you add interactivity to web pages.

HTML defines structure, CSS handles appearance, and JavaScript handles behavior like responding to user clicks, fetching data, or updating content.

Modern browsers expose a set of standardized APIs that JavaScript can use to interact with the page. This includes working with the DOM, making network requests, and much more.

To include javascript on the page, you can either inline it of link it.

### Accessing the DOM

You can use the `document` object (part of the browser’s window object) to access and modify the DOM.

Example:

```
const h1 = document.querySelector('h1');
 h1.style.color = 'red';

 h1.addEventListener('click', () => {
    if (h1.style.color === 'red') {
        h1.style.color = 'blue';
    } else {
        h1.style.color = 'red';
    }
 })
```

In this example, we:

1. Select the first `<h1>` using `document.querySelector('h1')`.
2. We set the color to red, using `style.color`.
3. Using `addEventListener` to listen `click` event, and provide a callback function which toggles the color.

> Note: These methods aren’t built into the JavaScript language — they come from the browser, which exposes them through the DOM API.

### Creating elements

```
const body = document.querySelector('body');

const p = document.createElement('p');
const text = document.createTextNode('new element');

p.appendChild(text);
body.appendChild(p);

```

Here, we:

1. Select the `body`.
2. Create a new `<p>` and a new text node.
3. Append the text node to the `<p>`, and the `<p>` to the `body`.

### JavaScript

At this point, you should have a completed card! However, we still haven't used any JavaScript. So next, we will remove the HTML we added, and instead add it through the DOM with JavaScript.

In the `card.html`, we have a `<script>` tag and a function called `ìnit`, where we can start adding our JavaScript. 

This is how we create a function, a block of code we wanna run, and assign it to a variable called `init`. All the function does so far is selecting an element with the id `app`, and saving it to a variable called `app`. 

```javascript

  const init = () => {
    const app = document.getElementById("app");
  };

```

This is how we call the above function, meaning we run whatever code we write in the `init` functions body.

```javascript

  init();

```

We will create the elements we need, using the `createElement` method on the `document` object. Lets start by creating the outer div and add the class `card` by adding the following code to the `init` function.

```javascript

  const card = document.createElement("div");
  card.classList.add("card");

```

We can repeat this pattern for all elements of the card, and create texts using the `createTextNode` method.

```javascript

  const picture = document.createElement("div");
  picture.classList.add("picture");

  const content = document.createElement("div");
  content.classList.add("content");

  const labelContainer = document.createElement("div");
  labelContainer.classList.add("label-container");

  const label1 = document.createElement("span");
  label1.classList.add("label");
  label1.classList.add("green");
  const label1Text = document.createTextNode("label 1");

  const label2 = document.createElement("span");
  label2.classList.add("label");
  label2.classList.add("red");
  const label2Text = document.createTextNode("label 2");

  const name = document.createElement("h2");
  name.classList.add("name");
  const nameText = document.createTextNode("name");

  const description = document.createElement("span");
  description.classList.add("description");
  const descriptionText = document.createTextNode("description");

```

Next, we need to append each element of the card to a parent:

```javascript

  label1.appendChild(label1Text);
  label2.appendChild(label2Text);
  labelContainer.appendChild(label1);
  labelContainer.appendChild(label2);

  name.appendChild(nameText);
  description.appendChild(descriptionText);

  content.appendChild(labelContainer);
  content.appendChild(name);
  content.appendChild(description);

  card.appendChild(picture);
  card.appendChild(content);

```

and append the card to the document:

```javascript

  app.appendChild(card);

```

### Bonus 

Although the code above works as expected, it is very verbose and it would be hard to understand what is happening at a first look. So maybe we can clean it up a little, to make it more readable. 

One pattern we use often is creating an `element` and assigning one or more classes to it. We could abstract this away to a utillity function, to avoid repeating ourselves, and make the code easier to read. 

```javascript 

  const createElemWithClasses = (element, ...classList) => {
    const elem = document.createElement(element);
    elem.classList.add(...classList);

    return elem;
  };

```

We could then call this function instead:

```javascript 

  const card = createElemWithClasses("div", "card");

```

Another common pattern is to create a `textNode` and append it to an `element`, we could create a helper for that as well.

```javascript

  const appendTextToElem = (text, element) => {
    const textNode = document.createTextNode(text);

    return element.appendChild(textNode);
  };

``` 

And call it like so: 

```javascript

  appendTextToElem("label 1", label1);

```
This way, our `init` function would then be a bit easier to read: 

```javascript 

  const init = () => {
    const card = createElemWithClasses("div", "card");
    const picture = createElemWithClasses("div", "picture");
    const content = createElemWithClasses("div", "content");
    const labelContainer = createElemWithClasses("div", "label-container");
    const label1 = createElemWithClasses("span", "label", "green");
    const label2 = createElemWithClasses("span", "label", "red");
    const name = createElemWithClasses("h2", "name");
    const description = createElemWithClasses("span", "description");

    appendTextToElem("label 1", label1);
    appendTextToElem("label 2", label2);
    appendTextToElem("name", name);
    appendTextToElem("description", description);

    labelContainer.appendChild(label1);
    labelContainer.appendChild(label2);

    content.appendChild(labelContainer);
    content.appendChild(name);
    content.appendChild(description);

    card.appendChild(picture);
    card.appendChild(content);

    const app = document.getElementById("app");
    app.appendChild(card);
  };

```

Or, to make it even easier to get an overview, we could create another function responsible for creating the card:

```javascript 

  const init = () => {
    const card = createCard();
    const app = document.getElementById("app");
    app.appendChild(card);
  };

```

If you are feeling comfortable so far, feel free to explore more ways to refactor the code. Just keep in mind that the point of refactoring should be readability and simplicity, not minimizing the lines of code.
